Теория к лабораторной работе:
Работа над проектами документов



Вниз

Прежде чем приступать к работе с документом, его необходимо ввести в систему, т.е. зарегистрировать. При регистрации формируется регистрационная карточка (РК) документа, в которую заносятся сведения о документе и присваивается регистрационный номер, формируемый по единым для данной подгруппы документов правилам.

Система «ДЕЛО» позволяет осуществлять регистрацию входящих, исходящих и внутренних документов: письма, приказы, договоры, обращения граждан и прочее.

При регистрации формируется регистрационная карточка документа, в которую заносятся сведения о документе и присваивается регистрационный номер, формируемый по единым для данной подгруппы документов правилам. Каждый тип РК имеет свой реквизитный состав, основанный на требованиях нормативных документов и делопроизводственной практике (у исходящих и внутренних документов он одинаков). К карточке могут быть прикреплены файлы, содержащие тексты или графические образы самих документов.

Электронная регистрационная карточка состоит из двух экранов. Первый экран содержит основные реквизиты РК, второй - дополнительные. Переход от раздела к разделу осуществляется с помощью закладок, расположенных в правом верхнем углу. В нижней части основного экрана отражаются относящиеся к этому документу поручения.

Система позволяет в автоматизированном режиме регистрировать переданные по электронной почте документы, в том числе снабженные электронной цифровой подписью. Для документов, которые готовятся с помощью MS Word, также реализована упрощенная процедура регистрации прямо из привычной для пользователя среды редактора.

Для облегчения и ускорения процедуры регистрации используются следующие инструменты:

  • задание автоматически заполняемых значений реквизитов (по умолчанию);
  • заполнение реквизитов путем выбора нужного значения из заранее настроенного справочника;
  • перечень наиболее часто используемых стандартных текстов;
  • установление ссылок на другие документы;
  • отнесение документа к тематическим рубрикам и др.

Задание:
1. Создайте в MS Word документ, содержащий информацию о работе организации в выходные дни (например, график дежурств).
2. В программе дело создайте регистрационную карточку проекта внутреннего документа, заполнив поля "Плановая дата", "Исполнитель", "Содержание", "Состав", "Версия", "Адресаты", "Файлы". В разделе "Управление проектом" укажите себя в качестве подписывающего и визирующего лица, а также в качестве визирующего лица укажите товарища, работающего на соседнем компьютере.
3. Отправьте проект документа на визирование.
4. Завизируйте свой проект и проект, присланный вам на визирование с соседнего компьютера.
5. Утвердите свой проект
6. Зарегистрируйте документ, созданный на основе утвержденного вами проекта.

Указания по выполнению работы:
Работа с проектами документов подразумевает выполнение следующих действий:
- создание регистрационной карточки проекта документа (РКПД);
- согласование проекта документа;
- утверждение проекта документа;
- регистрация готового документа, созданного на основе проекта.

1) Создание регистрационной карточки проекта документа:
а) Откройте папку "Управление проектами" функции "Кабинеты". Меню Управление - Создать РКПД.

б) В открывшемся окне из перечня групп документов выберите нужную. Нажмите "Выбрать".

Рис. 1 - Окно регистрационной карточки проекта документа

в) Заполните поля РКПД:
- "Плановая дата" - дата, к которой проект документа должен пройти все стадии подготовки проекта и стать подписанным документом;
- "Исп." - должностное лицо, являющееся исполнителем данного проекта документа. Значение выбирается из раскрывающегося списка. По умолчанию в поле устанавливается фамилия должностного лица, стоящего в этом перечне первым;
- "Содержание" - краткое содержание документа. Задается вводом с клавиатуры;
- "Адресаты" - перечень внешних или внутренних адресатов документа. Список внешних адресатов задается выбором значений из справочника "Организации" и/или "Граждане". Список внутренних адресатов задается выбором значений из справочника "Подразделения";
- "Состав" - состав документа. Задается вводом с клавиатуры (формат: количество листов документа);
- "Версия" - номер текущей версии проекта документа. Если существует несколько версий данного проекта, то для отображения нужной, следует выбрать ее номер из раскрывающегося списка;
- "Файлы" - список файлов, прикрепленных к РК проекта;
- В правом верхнем углу РКПД находится поле, в котором красным шрифтом указывается состояние проекта на текущий момент времени. Значение этого поля формируется автоматически и в зависимости от состояния проекта содержит разные формулировки.

В течение работы следите за измененииями в данном поле.

г) В разделе "Управление проектом" (см. рис.1) введите визирующих и/или подписывающих лиц: меню Действия - Визы/подписи, Добавить визирующего/подписывающего или кнопка на панели инструментов раздела:
- Из справочника "Подразделения" выберите должностных лиц, которые должны завизировать (подписать) данный проект и щелкните на кнопке Выбрать. Откроется окно, содержащее перечень выбранных должностных лиц.
- Из раскрывающегося списка, расположенного в левой верхней части окна, выберите одно из двух значений: "На визирование" или "На подпись".
- Щелкните на кнопке Добавить.

д) Отправьте проект на визирование/подписание: находясь в окне РКПД, выберите из меню Действия - Визы/подписи, Послать на визирование или Послать на подпись (или щелкните на кнопке на панели инструментов). Откроется окно, содержащее список должностных лиц, которые должны завизировать/подписать данный проект документа. Щелчком мыши выделите должностных лиц, которым требуется послать проект на визирование/подписание. Определив порядок рассылки (переключателей Всем сразу или По очереди) и продолжительность визирования/подписания (на … часов/дней), щелкните на кнопке ОК.

2) Визирование проекта документа:
а) В разделе "Управление проектом" в списке визирующих и подписывающих лиц выделите строку с вашей фамилией.

б) Выберите из меню Действия - Визы/подписи, Визировать или щелкните на кнопке, расположенной на панели инструментов раздела "Управление проектом".
Откроется окно, предназначенное для ввода и редактирования информации о визировании.

Рис. 2 - Окно визирования

В поле, расположенном в правой верхней части окна, автоматически устанавливается фамилия выбранного должностного лица. При необходимости, визирующего можно заменить, выбрав из раскрывающегося списка фамилию другого должностного лица.

в) В поле, расположенном в левой верхней части окна, укажите один из вариантов визы. Значение задается выбором из списка.

г) Укажите дату визирования. По умолчанию, в полях Дата и в установлены текущие дата и время, которые можно отредактировать.

д) Щелкните на кнопке ОК.
Окно ввода информации о визировании закроется, и в разделе "Управление проектом" в записи, относящейся к выбранному должностному лицу, автоматически будут установлены соответствующие значения в графах Виза\подпись и Дата:

Рис. 3 - Заполнение полей Визы/подписи и Дата

Сохраните РКПД: меню Файл - Записать РКПД или щелкните на кнопке .

е) Откройте РК проекта документа, подлежащего визированию в папке "На визирование" вашего кабинета (этот проект документа отправлен вам с соседнего компьютера). Завизируйте этот проект.

3) Утверждение проекта документа:
а) Откройте РК проекта документа, подлежащего утверждению и переведите его в режим редактирования: меню Действия - Редактировать РКПД

б) В разделе "Управление проектом" в списке визирующих и подписывающих лиц выделите строку с вашей фамилией.

в) Выберите из меню Действия - Визы/подписи, Подписать или щелкните на соответствующей кнопке, расположенной на панели инструментов раздела "Управление проектом".
Откроется окно, предназначенное для ввода и редактирования информации о подписании. При подписании проекта документа используется одно из двух фиксированных значений: Утверждаю или Не утверждаю. Утвердите проек.

4) Регистрация документа, созданного на основе утвержденного проекта:
а) При работе с РК проекта документа переведите РКПД в режим просмотра (щелкните на кнопке панели инструментов РКПД) и выберите из меню Файл - Регистрировать проект. Эту команду можно осуществить только если проект документа будет иметь статус подписан.
Откроется окно РК документа, относящегося к той же группе документов, что и проект. При этом часть реквизитов РК автоматически заполняется значениями, установленными в РКПД. ФИО и должности лиц, подписавших проект документа становятся значениями реквизита Подписал, а завизировавших проект - реквизита Визы. Файлы, прикрепленные к РКПД исполнителем проекта, прикрепляются к РК документа.

б) Введите недостающие реквизиты РК.

в) Завершите регистрацию документа, сохранив введенную информацию.
После регистрации документа, созданного на основе проекта, РКПД получает статус "Зарегистрирован" и удаляется из папки "Управление проектами" кабинета исполнителя проекта. Записи РКПД, оставленные в папке "На визировании" кабинета визирующих лиц, также удаляются из этой папки.
